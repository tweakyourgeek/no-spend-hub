## Existing Product Overview

### The 30-Day No-Spend Roadmap (index.html)
**Source**: Claude File 12
**Status**: BUILT AND LIVE
**URL**: https://tweakyourgeek.github.io/nospend-bus-roadmap/
**Format**: Single-page HTML, hosted on GitHub Pages (free)
**Built with**: Claude Code

**Purpose**: Primary wayfinding tool to prevent users from getting lost across multiple workbooks

### Technical Specifications

**Platform**: GitHub Pages (free hosting)
**File Type**: Single HTML file (CSS and JS embedded)
**Dependencies**: None (except fonts)
**Offline Capability**: Yes (works offline once loaded)
**Mobile**: Responsive and optimized for phone viewing
**Maintenance**: Easy to update (single file edit)

### Features

**1. Sticky Banner**
> "You can't do this wrong. Spending during the challenge doesn't mean failure—it means data."

**2. Three Pillar Boxes**
Quick links to:
- Workbook (explained as "The spine of the process")
- Journal (explained as "Reflection and awareness")
- Spreadsheet Pack (explained as "Optional infrastructure")

**3. "Where Am I?" Navigator**
Interactive buttons that scroll to appropriate section:
- I haven't started yet → Before You Start
- I'm in Week 1 (Days 1-7) → Week 1
- I'm in Week 2-3 (Days 8-21) → Weeks 2-3
- I'm in Week 4 or done (Days 22-30+) → Week 4 & Beyond
- I'm stuck/overwhelmed → Stuck? Start Here

**4. Four-Step Progression**
Collapsible sections for each phase:
- **Before You Start** - Prep & Setup
- **Week 1** - Noticing & Naming (Days 1-7)
- **Weeks 2-3** - Urges & Patterns (Days 8-21)
- **Week 4 & Beyond** - Integration & Next Steps (Days 22-30+)

**5. "Stuck? Start Here" Emergency Section**
Sage green background, highly visible
Scenarios addressed:
- Overwhelmed
- Spent money and feel bad
- Structure feels like too much
- Not sure which tool to use

**6. Functional Checkboxes**
- Save state to browser localStorage
- Track progress through the challenge
- Persist across browser sessions

**7. Navigation Features**
- Smooth scroll to sections
- Back to top button
- Collapsible sections

**8. Brand Integration**
Uses Tweak Your Geek colors:
- #3B3B58 (deep purple)
- #B375A0 (mauve)
- #A59880 (warm taupe)
- Sage green accents

### Distribution

**How it's used**:
- Linked in welcome emails
- Referenced on page 1 of Workbook and Journal
- Posted in Skool community
- Included in bundle partner materials
- Can be downloaded as standalone HTML file

**Versions**:
- **Web version** (primary): Interactive, collapsible, full context and support
- **Book version** (secondary): 1-page PDF in workbooks, minimal structure only, links to web version

## Integration Strategy

### How They Work Together

**Relationship**: index.html is the MAP, The Guide is the TERRITORY

**User Journey**:
1. **User purchases The Guide** (or gets access via Skool $27/mo tier)
2. **Welcome email includes link** to index.html roadmap
3. **Page 1 of The Guide** references roadmap: "Feeling lost? Visit the roadmap: [link]"
4. **User can toggle between**:
   - Roadmap for orientation ("Where am I? What's next?")
   - Guide for depth (daily prompts, worksheets, tracking)

### When to Use Each

**Use index.html when**:
- Just starting, not sure where to begin
- Feeling overwhelmed by materials
- Need quick reference to what's next
- Want to see big picture structure
- Accessing on phone/need mobile view
- Stuck and need emergency guidance

**Use The Guide when**:
- Working through daily prompts
- Doing deep reflection and journaling
- Filling out worksheets
- Tracking expenses
- Reading pattern explanations
- Reviewing reference materials

### Cross-Linking Strategy

**From index.html → The Guide**:
- Each collapsible section links to relevant Guide sections
- "Before You Start" links to Guide Section 1
- "Week 1" links to Guide Section 4 (Days 1-7)
- "Stuck? Start Here" links to Guide reference pages

**From The Guide → index.html**:
- Page 1: "Use the roadmap for navigation: [link]"
- Section breaks: "Lost? Check the roadmap"
- Reference pages: "Quick navigation: [roadmap link]"

**Unified Messaging**:
Both must consistently reinforce:
- "You can't do this wrong"
- Spending = data, not failure
- Multiple participation options valid
- Stakes stay light

## Recommendations

### Keep Both Products

**Why**:
- Different purposes (navigation vs depth)
- Different formats (web vs PDF)
- Different use cases (mobile reference vs deep work)
- Complementary, not redundant

**Don't**:
- Replace index.html with Guide
- Make them compete
- Duplicate content between them

### Update index.html Minimally

**Current Approach** (✅ Implemented):
- Keep week themes simple and generic (NOT framework-specific)
- No Lab Loop language (intentionally omitted)
- Links point to Skool community where materials are delivered
- Kept simple and oriented around wayfinding only

**Don't**:
- Add Aligned Life framework detail (that's Guide's job, not roadmap's)
- Add Chasing pattern language (already removed, keep it gone)
- Make it longer or more complex
- Remove existing emergency guidance ("Stuck? Start Here")

### Maintain Consistency

**Unified Elements**:
- Voice (conversational, direct, shame-free)
- Philosophy ("You can't do this wrong")
- Visual branding (colors, fonts)
- Key phrases and mantras

**Different Elements**:
- Depth (roadmap stays high-level)
- Interactivity (roadmap has buttons/navigation)
- Format (web vs PDF)
