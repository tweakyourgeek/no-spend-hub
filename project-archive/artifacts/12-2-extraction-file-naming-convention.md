# 16-Document Extraction File Naming Convention

- 01_Frameworks_And_Philosophy.md

## Document Descriptions

1. **Frameworks & Philosophy** - Core concepts, why this works, underlying principles, the Chasing Series framework
2. **Challenge Structure** - How the 30 days are organized, daily themes, progression, delivery model
3. **Participant Materials** - Worksheets, trackers, reflection prompts, daily content
4. **Skool Community Strategy** - Community structure, content placement, tier access, engagement plan
5. **Email/Communication Templates** - Daily emails, support messages, check-ins, onboarding
6. **Social Media Content** - Posts, captions, engagement ideas
7. **Facilitator/Admin Guide** - How to run the challenge, facilitate community, handle issues
8. **Checklists & Tools** - Setup checklists, tracking systems, resources
9. **Strategic Insights** - Marketing approach, positioning, pricing discussions, bundle strategy
10. **Brand Voice Guide** - Tone, style, language patterns, what to avoid
11. **Product Architecture** - Tiers, pricing, deliverables per tier, upgrade paths, FREE vs PAID structure
12. **Evolution Documentation** - Track how the product evolved from PLR through Content Queen to current form
13. **Parked Ideas** - Ideas mentioned but not implemented, "someday" features, future possibilities
14. **Cross-References to Existing Product** - Compare conversations to the existing 30-Day No Spend Roadmap, note what's built vs. what's only discussed
15. **Master Index** - Cross-references and navigation across all documents
16. **Gaps & Contradictions** - What's unclear, what's missing, what conflicts

## Output Requirements

**For Each Document:**
- Clear hierarchical headers
- Preserve source attribution (which conversation file)
- Quote exact language when important (especially for verbally processed insights)
- Flag items that need clarification
- Note contradictions or multiple versions
- Be comprehensive (don't summarize away important details)
- **Preserve the evolution** - show how ideas changed over time, don't just give final version

## What NOT To Do

- Don't summarize away important details
- Don't assume things are the same (extract what's actually there)
- Don't skip contradictions (flag them)
- Don't combine different evolutionary versions (show progression)
- Don't ignore "quick conversations" (they may contain key insights)
